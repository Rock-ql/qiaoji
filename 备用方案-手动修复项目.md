# 备用方案：手动修复Xcode项目配置

如果移除并重新添加Statistics文件夹后还是报错，说明可能是Xcode项目文件损坏。

## 方案1：检查项目文件

1. 在Finder中找到你的Xcode项目文件（.xcodeproj）
2. 右键点击 → 显示包内容
3. 用文本编辑器打开 `project.pbxproj`
4. 按 Cmd+F 搜索 "StatisticsView"
5. 检查是否有重复的引用

## 方案2：创建新的Xcode项目

如果项目文件确实损坏，可以：

1. 创建一个全新的Xcode项目
2. 将整个 ExpenseTracker 文件夹拖入新项目
3. 配置项目设置（Bundle ID、Team等）

## 方案3：临时绕过（不推荐）

在ContentView.swift的顶部添加：

```swift
import SwiftUI
import SwiftData

// 移除这个导入如果有
// import Statistics
```

然后完全限定StatisticsView的路径。

## 联系支持

如果以上方案都不行，这可能是Xcode的bug。请：
1. 重启Mac
2. 更新Xcode到最新版本
3. 检查macOS是否有待更新
